# 편입영어 AI 튜터
## 프로토타입 기획서 v1.0

> Claude API 기반 · 편입영어 문법 특화 · 개인 맞춤 학습

---

## 1. 프로젝트 개요

### 1.1 한 줄 정의

Claude AI가 실시간으로 문제를 생성하고, 학생의 약점을 진단하여 맞춤 커리큘럼과 일일 학습을 자동으로 운영하는 편입영어 전문 AI 튜터 앱

---

### 1.2 핵심 차별점

| 기존 앱의 한계 | 본 앱의 강점 |
|---|---|
| 회화·스피킹 특화, 한국 수험 대응 없음 | 편입영어 문법 파트 완전 특화 |
| 문법 문제 생성+채점 기능 없음 | AI 실시간 문제 생성 + 즉시 채점 |
| 커리큘럼 자동 생성 없음 | 진단 → 커리큘럼 → 일일학습 자동화 |
| 편입/공무원/수능 파트 특화 없음 | 전문 강사 노하우를 프롬프트에 내재화 |
| 발전 리포트 수준 낮음 | 주간/월간 성장 리포트 자동 생성 |

---

### 1.3 프로토타입 범위

**대상 시험:** 편입영어 — 문법 파트 (관계사, 분사구문, 가정법, 시제, 수일치, 병렬구조 등)

나머지 시험(공무원, 수능)과 파트(어휘, 독해, 논리독해)는 구조 동일하게 확장 예정.
프로토타입에서 구조만 잡으면 콘텐츠 교체로 대응 가능.

---

### 1.4 콘텐츠 생성 전략

- **Claude API 기본 성능** — 편입영어 수준 문법 문제 즉시 생성 가능
- **강사 자료 보강 (RAG)** — 와이프 보유 자료(PDF, 노트)를 업로드해 시스템 프롬프트에 주입
- **시스템 프롬프트 핵심 내용:**
  - 대학별 출제 경향 (건국대, 홍익대, 중앙대, 경희대 등)
  - 자주 출제되는 함정 패턴
  - 난이도 기준 상/중/하 정의
  - 오답 해설 방식 (어떤 톤, 어떤 구조로 설명할지)
- **프로세스:** AI 생성 → 강사 검수 → 프롬프트 개선 → 반복

## 3. 화면별 상세 기획

### SCREEN 01 — 온보딩
*학생 정보를 수집하고 개인화 기반을 세팅하는 첫 화면*

- 시험 유형 선택 버튼: `[편입영어]` `[공무원영어]` `[수능영어]` (프로토타입은 편입영어만 활성)
- 목표 시험일 입력 → D-day 자동 계산 → 남은 주 수 산출
- 하루 학습 가능 시간 선택: 30분 / 1시간 / 2시간 / 3시간 이상
- 목표 대학 선택 (선택): 건국대, 홍익대, 중앙대, 경희대, 기타 → 시스템 프롬프트 자동 분기
- 입력 완료 → 레벨 테스트 화면으로 이동

---

### SCREEN 02 — 레벨 테스트
*AI가 실시간으로 문제를 생성하고 파트별 실력을 진단*

- 총 24문항 / 파트별 4문항씩 (관계사, 분사구문, 가정법, 시제, 수일치, 병렬구조)
- Claude API 호출로 문제 실시간 생성 — DB 불필요
- 4지선다 형식 / 답 선택 시 다음 문제 자동 이동
- 문제 전체 완료 후 `[채점하기]` 버튼 → 즉시 채점
- 결과값: 파트별 정답수, 소요시간, 오답 번호 배열 → 진단 리포트로 전달
- **주의:** 레벨 테스트 단계에서는 해설 미표시 (동기 저하 방지)

---

### SCREEN 03 — 진단 리포트 + 커리큘럼
*진단 결과를 시각화하고 맞춤 학습 로드맵을 자동 생성*

- 레이더 차트: 6개 파트별 점수를 방사형으로 시각화
- 강점 파트 (70점 이상) / 보통 (40~69점) / 취약 파트 (39점 이하) 색상 구분
- AI 진단 코멘트: 학생 수준을 3~4문장으로 요약 (Claude API 생성)
- 주차별 커리큘럼 자동 생성:
  - 취약 파트에 학습 시간 가중 배분
  - 목표 시험일 D-day 역산으로 주차 구성
  - 하루 공부시간 기반 일일 문제 수 자동 결정
- 커리큘럼 확정 버튼 → 일일 학습 대시보드로 이동

---

### SCREEN 04 — 일일 학습 대시보드
*매일 반복되는 학습 사이클의 핵심 화면*

- `[오늘의 문제]` 탭 — AI가 오늘 파트에 맞는 문제 생성 (커리큘럼 기반)
- `[오답 노트]` 탭 — 이전에 틀린 문제 재출제 + 상세 해설
- `[오늘의 단어]` 탭 — 파트 연관 핵심 어휘 5~10개 + 예문
- `[숙제 체크]` 탭 — 오늘 완료해야 할 미션 목록 + 완료 체크
- 진도 바: 오늘 목표 문제 수 대비 완료율 시각화
- **문제 풀이 UX:**
  - 답 선택 즉시 정오답 표시
  - 오답 시: 왜 틀렸는지 상세 해설 (Claude API 생성)
  - 오답 시: 헷갈리기 쉬운 함정 포인트 별도 표시
  - 오답 시: 유사 문제 1개 추가 출제 (약점 보강)
- 일일 완료 시 `[오늘 학습 완료]` 버튼 → 간단한 성취 화면

---

### SCREEN 05 — 발전 리포트
*주간 성장을 시각화하고 다음 주 학습 방향을 제안*

- 주간 점수 변화 그래프 (파트별 꺾은선 차트)
- 이번 주 총 학습량: 문제 수 / 정답률 / 학습 일수
- 가장 많이 틀린 문제 유형 Top 3 분석
- 레벨업 테스트: 4주 단위로 전체 레벨 재진단 → 커리큘럼 자동 조정
- **AI 주간 리포트 코멘트 (Claude API 생성):**
  - 이번 주 잘한 점 / 아쉬운 점
  - 다음 주 집중 파트 제안
  - 동기 부여 메시지 포함
- 다음 주 커리큘럼 미리보기 + 수정 옵션

## 4. 학습 사이클 상세

### 4.1 하루 학습 구조

| 시간 | 활동 | 문제 수 | 비고 |
|---|---|---|---|
| 0~5분 | 워밍업 복습 | 오답 3문항 | 전날 오답 재출제 |
| 5~25분 | 오늘의 신규 문제 | 10~15문항 | 커리큘럼 파트 집중 |
| 25~30분 | 오늘의 단어 | 5~10개 | 파트 연관 어휘 |
| 30분~ | 추가 (선택) | 취약 파트 추가 | 학습 여유 있을 때 |

---

### 4.2 오답 처리 프로세스

답 선택 즉시 정오답 표시.

오답인 경우 — 3단계 해설 제공:

1. **1단계** — 정답 + 한 줄 핵심 설명 (즉시 표시)
2. **2단계** — 상세 문법 해설 (왜 정답인지, 나머지 보기가 왜 오답인지) — 펼치기
3. **3단계** — 함정 포인트 (수험생이 자주 틀리는 이유 + 주의사항) — 별도 강조

오답 문항은 오답 노트에 자동 저장.
다음날 워밍업 복습 세션에서 재출제.
같은 문항을 **2회 이상 틀리면** 해당 파트 집중 학습 모드 자동 전환.

### 4.3 레벨업 테스트 주기

| 테스트 | 주기 | 내용 |
|---|---|---|
| 미니 레벨 체크 | 매주 (일요일) | 이번 주 학습 파트 10문항 → 다음 주 커리큘럼 조정 |
| 전체 레벨 테스트 | 4주마다 | 전 파트 24문항 → 전체 커리큘럼 재설정 |
| D-30 모의고사 | 시험 30일 전 | 실전 풀세트 → 취약점 최종 점검 |

## 5. AI 콘텐츠 생성 구조

### 5.1 Claude API 역할 분담

| 기능 | Claude API가 하는 일 |
|---|---|
| 문제 생성 | 파트, 대학 스타일, 난이도 인풋 → JSON 형식 문제 출력 |
| 채점 + 해설 | 정오답 판단 + 3단계 해설 + 함정 포인트 생성 |
| 진단 코멘트 | 레벨테스트 결과 → 학생 수준 분석 3~4문장 |
| 커리큘럼 생성 | 약점 파트 + D-day + 하루 학습시간 → 주차별 로드맵 |
| 일일 숙제 생성 | 오늘 파트 + 오답 이력 → 오늘의 문제 세트 구성 |
| 주간 리포트 | 한 주 데이터 → 잘한 점/아쉬운 점/다음 주 제안 |
| 단어장 | 오늘 학습 파트 연관 핵심 어휘 + 예문 생성 |

---

### 5.2 강사 자료 보강 방식

- **1단계 (프로토타입)** — 강사가 파트별 출제 경향, 함정 유형을 텍스트로 정리 → 시스템 프롬프트에 직접 삽입
- **2단계 (고도화)** — PDF/노트 파일 업로드 → RAG 방식으로 API 호출 시 자동 참조
- **3단계 (장기)** — 충분한 데이터 축적 후 Fine-tuning 검토

> **핵심 원칙:** 기술은 Claude API가 담당, 퀄리티는 강사 노하우가 결정. 강사가 '이 부분이 실제 시험과 다르다'고 피드백 → 프롬프트 개선 → 반복.

### 5.3 문제 생성 JSON 구조

```json
{
  "id": 1,
  "type": "관계사",
  "question": "다음 빈칸에 알맞은 것을 고르시오.",
  "sentence": "The professor _____ lecture I attended was very inspiring.",
  "options": ["(A) who", "(B) whose", "(C) whom", "(D) which"],
  "answer": "(B) whose",
  "explanation": "선행사가 사람(The professor)이고 빈칸 뒤에 명사(lecture)가 이어지므로 소유격 관계대명사 whose가 정답입니다. who는 주격, whom은 목적격으로 뒤에 명사가 올 수 없습니다.",
  "trap": "who와 whose를 혼동하기 쉬운 문항. 빈칸 뒤에 명사가 오면 반드시 소유격(whose)임을 확인할 것.",
  "difficulty": "중급"

```

### 7.2 오답 재출제 로직

| 조건 | 처리 |
|---|---|

## 8. 개발 순서 (바이브코딩 로드맵)

| 단계 | 기간 | 작업 내용 |
|---|---|---|
| Phase 1 | 1~2주 | 프로젝트 세팅 + Claude API 문제 생성 + 채점 동작 확인 |
| Phase 2 | 3~4주 | 레벨 테스트 화면 + 진단 리포트 + 커리큘럼 생성 화면 |

## 9. 강사 협업 포인트

### 강사가 제공해야 할 정보

- 파트별 핵심 문법 포인트 목록 (관계사: 어떤 유형이 자주 나오는지)
- 대학별 출제 경향 요약 (건국대 패턴, 홍익대 패턴 등)
- 자주 등장하는 함정 유형 목록 + 수험생이 헷갈리는 이유
- 난이도 기준 정의 (초급/중급/고급 각각 어떤 수준의 문장인지)
- 해설 선호 방식 (어떤 톤, 어떤 구조로 설명해야 학생이 잘 이해하는지)

### 피드백 사이클

1. AI 생성 문제 20개를 강사가 검토 → "이 문제는 실제 시험과 다름" 피드백
2. 피드백 내용을 시스템 프롬프트에 반영 → 재생성
3. 반복 (보통 3~5사이클이면 퀄리티 안정화)

> **목표 기준:** 강사가 AI 생성 문제를 보고 "이거 내가 직접 만들었어도 이 수준은 됐겠다"는 반응이 나오면 프롬프트 완성.
